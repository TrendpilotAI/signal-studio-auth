## CRITICAL

### C1 — Missing Rate Limiting on Auth Routes
**File:** `routes/auth_routes.py` — `/auth/login` (~line 71), `/auth/signup` (~line 52)
**Issue:** No rate limiting. Any IP can brute-force credentials or spam signups.
**Fix:** Add `slowapi` with Redis backend. Limit login to 5 req/min per IP, signup to 10/hr.

### C2 — Hardcoded Fallback JWT Secret
**File:** `config/supabase_config.py` — line 35
**Issue:** `FORWARDLANE_JWT_SECRET = os.environ.get("AUTH_SECRET_KEY", "very_secure_secret")`
A missing env var silently falls back to a known weak secret, allowing forged tokens in prod.
**Fix:** Raise `ValueError("AUTH_SECRET_KEY must be set in production")` when env var is absent.
Similarly, `SUPABASE_JWT_SECRET` defaults to `""` — an empty string causes PyJWT to accept ANY token.
**Fix:** Add startup validation: `if not SUPABASE_JWT_SECRET: raise RuntimeError(...)`.

## HIGH

### H1 — invite-to-org Missing Admin Role Check
**File:** `routes/auth_routes.py` — `invite_to_org()` ~line 113
**Issue:** Checks `caller.is_authenticated` but NOT `caller.role == "admin"`. Any authenticated user
can invite anyone to any organization.
**Fix:** Add `if getattr(caller, 'role', None) not in ('admin', 'owner'): raise HTTPException(403)`.

### H2 — No Password Reset / Forgot Password Route
**File:** `routes/auth_routes.py` (missing entirely)
**Issue:** No `/auth/reset-password` or `/auth/forgot-password`. Login flow is incomplete for prod.
**Fix:** Add proxy for `POST /auth/v1/recover` (send reset email) and `PUT /auth/v1/user` (update password).

### H3 — No User Migration Script
**File:** (missing entirely)
**Issue:** No tooling to migrate existing ForwardLane users to Supabase. Dual-mode cutover is blocked.
**Fix:** Write `scripts/migrate_users.py` — reads FL Postgres, creates Supabase users via admin API,
sets `app_metadata.legacy_user_id`. Must be idempotent (skip existing emails).

### H4 — httpx AsyncClient Created Per Request
**File:** `routes/auth_routes.py` — every route handler, ~lines 54, 73, 95, 108, 119
**Issue:** `async with httpx.AsyncClient() as client:` inside each handler creates/destroys a new
connection pool on every request. Under load this causes connection exhaustion and latency spikes.
**Fix:** Use FastAPI lifespan to create a module-level `AsyncClient` with connection pooling.
