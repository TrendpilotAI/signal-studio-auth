## HIGH

### [HIGH-1] New httpx.AsyncClient Per Request (No Connection Pooling)
**Files:** `routes/auth_routes.py` lines 113, 128, 144, 155, 175, 195  
**Issue:** Every route handler creates `async with httpx.AsyncClient() as client:` — opens a new TCP connection to Supabase on every request. This adds 10-50ms latency per call and wastes file descriptors under load.  
**Fix:** Create a module-level `AsyncClient` instance with lifespan management:
```python
# In main.py lifespan:
@asynccontextmanager
async def lifespan(app: FastAPI):
    app.state.http = httpx.AsyncClient(timeout=10.0, limits=httpx.Limits(max_connections=100))
    yield
    await app.state.http.aclose()
```
**See:** TODO-359

### [HIGH-2] In-Memory Rate Limiter Lost on Restart
**File:** `routes/auth_routes.py` lines 25-52  
**Issue:** `_RateLimiter._calls` is a module-level dict — reset on every process restart. Attacker can exhaust rate limit, wait for deploy/restart, and start fresh. Doesn't work across replicas.  
**Fix:** Redis sliding-window rate limiter  
**See:** TODO-402

### [HIGH-3] No Test Coverage for Core Auth Routes
**File:** `tests/` — only `test_security.py` exists  
**Issue:** No tests for `/me`, `/logout`, `/refresh`, `/invite-to-org`, nor for the dual-mode middleware switching logic or the ForwardLane fallback path.  
**Missing test scenarios:**
- Valid JWT → middleware populates `request.state.user`
- Expired JWT → 401 response
- ForwardLane fallback in dual mode
- `/me` returns correct user dict
- `/logout` invalidates session
- `/refresh` returns new tokens
**See:** TODO-357

### [HIGH-4] Pydantic v1 Compat Code in _compat.py
**File:** `middleware/_compat.py` lines 30-32  
**Issue:** `class Config` with `allow_population_by_field_name = True` and `orm_mode = True` are Pydantic v1 syntax. Pydantic v2 silently ignores unknown Config keys, meaning `User.model_dump(by_alias=True)` may not work correctly.  
**Evidence:** `routes/auth_routes.py` line ~165: `return user.dict(by_alias=True)` — `.dict()` is deprecated in v2.  
**Fix:** Migrate to `model_config = ConfigDict(from_attributes=True, populate_by_name=True)`  
**See:** TODO-404

## MEDIUM

### [MEDIUM-1] No CORS Middleware
**File:** No `main.py` or app factory visible in repo  
**Issue:** FastAPI doesn't add CORS headers by default. Signal Studio frontend (different origin) will get CORS errors.  
**Fix:** Add `CORSMiddleware` with `CORS_ORIGINS` env var  
**See:** TODO-405

### [MEDIUM-2] Synchronous JWT Decode in Async Context
**File:** `middleware/supabase_auth_middleware.py` line 47  
**Issue:** `pyjwt.decode()` is CPU-bound/synchronous but called in an async FastAPI middleware. Under high load, this blocks the event loop.  
**Fix:** Wrap in `asyncio.get_event_loop().run_in_executor(None, pyjwt.decode, ...)` or use `anyio.to_thread.run_sync()`  
**Severity:** MEDIUM (only matters at scale)

### [MEDIUM-4] _uuid_to_int Hash Collision Risk
**File:** `mapping/user_mapping.py` lines 54-58  
**Issue:** `_uuid_to_int` takes last 8 hex chars of UUID → 32-bit int. Collision probability ~1 in 4 billion — unlikely but non-zero. If two UUIDs map to same int, users get wrong org permissions.  
**Fix:** Store numeric ID in `app_metadata.legacy_user_id` for all users (migration script), not fallback hash. Treat hash fallback as error case that logs a warning.

### [MEDIUM-5] No Error Tracking in Production
**Issue:** Auth failures, JWT errors, and Supabase timeouts are logged to stderr only. No alerting, no aggregation.  
**Fix:** Add Sentry integration  
**See:** TODO-405
