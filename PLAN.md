## Architecture Overview

`signal-studio-auth` is a **thin auth proxy** between Signal Studio clients and Supabase. It adds:
1. Opaque refresh token layer (hides Supabase refresh tokens from clients)
2. RBAC enforcement via JWT claims
3. Rate limiting (Redis + in-memory fallback)
4. Security headers

### Current Architecture
```
Client → signal-studio-auth (FastAPI)
                │
                ├─ Redis (token store + rate limits)
                │
                └─ Supabase (identity provider)
```

### Proposed Architecture Additions
```
Client → signal-studio-auth (FastAPI)
                │
                ├─ Redis (tokens + rate limits + org membership cache)
                ├─ Supabase (identity provider)
                ├─ Sentry (error tracking)
                ├─ Structured log → Datadog/Grafana
                └─ Slack (security alerts)
```
